## 1. 개요

케미체크(ChemiCheck)는 가정용 생활화학제품의 성분을 카메라로 촬영하면 즉시 안전성을 분석해주는 iOS 앱입니다. 앱의 핵심 기능은 **실제 정부 공공데이터**에 기반합니다. 하드코딩된 예시 데이터 없이, 아래 기관의 오픈데이터를 직접 파싱·가공하여 앱에 탑재했습니다.

| 활용 기관 | 데이터 성격 |
|----------|-----------|
| 기후에너지환경부 화학물질안전원 | 화학물질 독성·위험 정보 |
| 환경부 (한국환경산업기술원, KEITI) | 생활화학제품 등록·전성분 공개 정보 |
| 식품의약품안전처 | 의약품·화장품 회수 알림 |

**Step 2 — 위험도 자동 분류 알고리즘**

증상 텍스트의 키워드를 분석하여 1~5단계 위험도를 자동 부여합니다.

```python
def risk_from_symptoms(general: str, inhale: str) -> int:
    text = general + inhale
    if "치명적" or "사망" or "발암" or "폭발" in text: return 5  # 매우위험
    if "독성"  or "폐부종" or "화상" or "신장손상" in text: return 4  # 위험
    if "자극"  or "메스꺼움" or "두통" or "기침" in text:    return 3  # 경고
    if text is empty or "자료없음":                          return 1  # 안전
    return 2  # 주의 (기본)
```

## 5. 앱 내 기능별 데이터 활용

### 5-1. OCR 성분 인식 (LocalDBService + OCRService)

**관련 파일:** `ChemiCheck/Services/OCRService.swift`, `ChemiCheck/Storage/LocalDBService.swift`

카메라로 제품 라벨을 촬영하면 Apple Vision Framework가 텍스트를 추출하고, 추출된 텍스트에서 화학물질명을 찾는 과정에 화학물질안전관리정보 DB가 사용됩니다.

```
[카메라 촬영] → [VNRecognizeTextRequest OCR] → [parseChemicalNames()]
                                                    ↓
                 ┌──────────────────────────────────────────────────┐
                 │ 0차: LocalDB 7,189건 전체 이름 메모리 조회         │
                 │      한국어명 포함 검사 → 영어명 포함 검사 순으로  │
                 ├──────────────────────────────────────────────────┤
                 │ 1차: 수동 alias 사전 (84개 표준 화학물질 + 별칭)   │
                 ├──────────────────────────────────────────────────┤
                 │ 2차: 한국어 패턴 정규식                           │
                 │      ([가-힣]+나트륨, [가-힣]+산, 등)              │
                 ├──────────────────────────────────────────────────┤
                 │ 3차: 영문 패턴 정규식                             │
                 │      (sodium [a-z]+, [a-z]+ chloride 등)          │
                 └──────────────────────────────────────────────────┘
                                        ↓
                 [인식된 화학물질명] → DB에서 위험도·증상 정보 조회
```

**앱 시작 시 동작:**

## 6. 데이터 품질 처리 방법

### 6-1. 인코딩 처리

| 문제 | 해결 방법 |
|------|----------|
| CSV BOM (UTF-8 with BOM) | `encoding="utf-8-sig"` 지정 |
| XLSX 다중 시트 | 시트명 직접 지정 (`wb["시트명"]`) |
| XLSX 헤더 오프셋 | `min_row=2` 또는 `min_row=3` 조정 (일부 파일이 1행 비어있음) |

### 6-2. 데이터 정제

| 문제 | 해결 방법 |
|------|----------|
| 물질명 선행 `·` 문자 | `clean()` 함수로 제거 |
| None/NULL 값 | 빈 문자열로 대체 |
| 증상 텍스트 초장문 | 길이 제한 (일반증상 600자, 흡입 400자, 나머지 300자) |
